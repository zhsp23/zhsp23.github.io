Kotlin学习笔记6-可见性修饰符

发表于 2018-02-25 | 分类于 Kotlin_android |

可见性修饰符

Kotlin中这些修饰符是与我们Java中的使用是有些不同的。在这个语言中默认的修饰符是 public ，这节约了很多的时间和字符。但是这里有一个详细的解释关于在Kotlin中不同的可见性修饰符是怎么工作的。

修饰符

private

private 是我们使用的最限制的修饰符。它表示只能被自己所在的文件可见，所以如果给一个类生命为private，就不能在定义这个类之外的文件使用它。

另一方面，如果在一个类类面使用了private 修饰符，那访问权限就被限制在这个类里面。甚至继承这个类的子类也不能使用它。

所以一等公民，类、对象、接口。。。。。。（也就是包成员）如果被定义为private，那么它们只会被定义所在的文件可见。如果被定义在了类或者接口中，那么它们只对这个类或者这个接口可见。

protected

这个修饰符只能用在类或者接口中的成员上。一个包成员不能被定义为 protected 。定义在一个成员中，它就与java中的方式一样了：它可以被成员自己和继承它的成员可见（比如，类和它的子类）

internal

如果是一个定位为 internal 的包成员的话，对所在的整个 model 可见。如果它是一个其他领域的成员，它就需要依赖那个领域的可见性。比如：一个类被定义为 private , 那么它的 internal 修饰的函数的可见性就会限制与它所在的这个类的可见性了。

我们可以访问 module 中的 internal 修饰的类，但是不能访问其它 module 的。

什么是module

根据Jetbrains的定义，一个 module 应该是一个单独的功能性的单位，它应该是可以被单独编译、运行、测试、debug的。根据我们项目不同的模块，可以在Android Studio中创建不同的 module 。在Eclipse中，这些 module 可以认为是在一个 workspace 中的不同的 project 。

public

这是最没有限制的修饰符，并且是默认的修饰符，成员在任何地方被修饰为 public ，很明显它只局限与它的领域。一个定义为 public 的成员被包含在一个修饰为private的类中，这个成员在这个类外也是不可见的。
